### 2.1 唐亦安-我是如何走出新手村的

![唐亦安](/images/blog/xianxiahuodong/gefei-shenzhen-share-20251213/tangyian.jpg)

这个嘉宾在今年下半年哥飞的每月新词比赛中持续霸榜，他做的主要内容是游戏攻略站/工具站。

他还分享了监控 sitemap 的工具，配置了飞书机器人的webhook之后，可以每天定时发送到飞书里：  
[https://github.com/taingh/sitemap-monitor](https://github.com/taingh/sitemap-monitor)

![监控工具](/images/blog/xianxiahuodong/gefei-shenzhen-share-20251213/tangyian-sitemap-tools.png)

#### 阶段一：游戏站
初入陌生领域，不知如何做起：按照社群内教程、按部就班

尝试过的找词方法：Google trends词根，大站子域名，监控大站sitemap，刷 hugging face，社媒热点

游戏站很适合新手：没什么门槛，但竞争愈发激烈，卷找词、卷上站速度、卷外链

![阶段一](/images/blog/xianxiahuodong/gefei-shenzhen-share-20251213/tangyian-part1.png)

#### 阶段二：游戏攻略/工具站
游戏攻略站具体操作方法：

+ 大流量平台：google play、apple store、stream、roblox知名游戏厂商
+ 避开大体量游戏：有官网（吃掉主词流量）、有专门团队做相关站点（竞争不过）；如任天堂、PS、英雄联盟
+ 通过 trend/社媒：发现即将发布/刚刚发布的游戏
+ 根据 youtube、tiktok视频播放量、discords社群人数，判断是否值得做
+ 根据游戏内容，规划网络结构：
    - 网站主页-承接游戏名主词流量
    - 网站内页-承接长尾词流量，如游戏名+人物、游戏名+道具
    - 只要是有用户搜索的词、有一定流量的词，就要单独做一个页面承接流量
+ 网站内容来源：自己去游戏内体验，YouTube攻略视频、游戏官方发布的数据

![阶段二](/images/blog/xianxiahuodong/gefei-shenzhen-share-20251213/tangyian-part2.png)

#### 阶段三：AI工具站
实用建议：

+ 模板的重要性：随着每次上站逐步完善模板
+ 关于网站部署：
    - 没有技术背景：推荐cloudflare全家桶，省心省事，不需要维护
    - 有技术背景：推荐 dokploy
+ 确定哪些是上站之前就要做好的：网站结构必须提前规划好，上线后SEO内容尽量不要修改
    - 网站结构 TDH SEO内容
    - 绑定 aherf
    - 上站前做基础的检查
+ 确定哪些是上站之后可以优化的：
    - 多语言
    - 增加内页
    - UI动画
+ 使用工具：推荐使用NetGlow扫描基础SEO问题
+ 社群交流：与朋友多交流，能获得额外收获

#### 总结
+ 新手按社群标准教程执行，不要过度思考
+ 尝试多种找词方法，包括词根、子站域名、监控大站sitemap等
+ 通过社交媒体关注更多相关博主，利用推荐机制发现新游戏
+ 建立并使用自己的网站模板，提高效率
+ 提前规划网站结构，一个搜索词对应一个页面
+ 使用NetGlow扫描网站基础SEO问题

### 2.2 卡叽哇-我的出海之路
![咔叽哇](/images/blog/xianxiahuodong/gefei-shenzhen-share-20251213/kajiwa.jpg)
这老哥在6月的年中上海分享会上已经见过了，当时的分享燃爆全场了，这次是作为开场嘉宾，依然是我的榜样和楷模。

#### 独立开发的劣势与优势
劣势：

+ 精力有限：
    - 一天就24小时，你要自己写代码、架构、运维、设计、客服、运营
    - 解法：将事情变成"模板+流程"，避免重复造轮子
+ 能力有限：
    - 开发、设计、SEO、SEM等很难同时达到专业水平
    - 解法：主攻SEO，其他能力靠AI补足或花钱解决。
+ 资源有限：
    - 既拼不了投放渠道，也撑不起长期试验
    - 解法：做已经被验证的需求，更小更精准的切入

优势：

+ 成本低，决策快
    - 不用开会和排期，今晚想做今晚就能上线
+ 灵活性极强
    - 新模型、新API出来，可以马上试水，抢前期红利
+ 收益高度归属个人
    - 所有投入都沉淀在你自己身上，失败也在累积资产

#### 哪些适合做独立开发？哪些不适合？
不适合：重运营、重内容、要大规模客户/审核、严重侵占精力的产品

适合：需求刚性、功能单一、技术可控、能靠SEO获取用户的轻量型工具产品

![独立开发](/images/blog/xianxiahuodong/gefei-shenzhen-share-20251213/kajiwa-solodev.png)

#### AI带来的变革
+ 改变了搜索行为，用户会在搜索中加上"AI"关键词
+ 技术民主化，低成本使用大模型
+ 思考如何让AI"打工"，为你创造收益

#### 关于需求挖掘/找词
什么是好的需求/关键词：

+ 长期：需求能持续几年，不是一阵风的热点
+ 稳定：底层服务可靠，API 供应和成本可控
+ 低风险：合规可持续，不碰灰黑产、不踩红线

如何寻找需求/关键词：

+ 数据平台精确挖词：使用SEMrush、SimilarWeb等，从词根出发挖相关站点和长尾需求
+ 从社媒案例反推需求：使用场景/实际案例 -> 谷歌上找到对应的站 -> 提炼核心关键词
+ 跟踪行业头部网站动向：新上的功能页->提炼核心关键词->判断流量和商业价值
+ 查看导航站：看看最近新上线的网站解决什么需求

#### 关于外链
外链哪里来：

+ 1.看竞品外链：看竞品的网站都在哪些地方投放、挂链接，列个清单
+ 2.主动上导航站：向行业头部导航站、工具目录站主动提交自己的网站
+ 3.深挖现有流量来源：从自己当前的流量来源里，找出已经带来访客的外链站点，想办法深度合作
+ 4.用搜索结果找新外链位：搜索相关关键词，在 SERP 前几页里寻找可以提交站点、写介绍的博客和社区

什么是好的外链：

+ 1.流量：能带来真实流量，只要持续带来相关用户，在我这里就是顶级外链
+ 2.品牌（有权重）：核心搜索词是品牌
+ 3.网站是活跃的：site 网站，最近7天有没有收录
+ 4.网站运营时间久：查看网站的注册时间

![外链](/images/blog/xianxiahuodong/gefei-shenzhen-share-20251213/kajiwa-backlinks.png)

#### 关于网站SEO生命周期
+ 起步阶段：靠外链和推广点火，第一批用户来自社区发帖、导航站、产品榜、朋友转发、少量广告
+ 试用阶段：当有人开始使用你的网站时，谷歌会给新站一点试验性的曝光，看点击率和留存时间
+ 爬升阶段&稳定阶段：表现好就逐渐往前排，最后在一批关键词上形成相对固定的排名和稳定自然流量

#### 成功后的建议
+ 做好备份方案，预防危机
+ 在账号、资金、基础设施上隔离高风险项目和稳定收入项目
+ 当基础稳定后，尝试更有挑战性的项目

#### 总结
+ 模块化工作流程，利用AI提高效率
+ 专注个人专长领域，对其他能力寻求AI辅助或外包
+ 评估并优化现有外链质量
+ 为成功项目建立备份和风险隔离机制
+ 持续更新维护网站以保持竞争力

### 2.3 空弦Uni-独立开发者的出海方法论与实践路径
![空弦](/images/blog/xianxiahuodong/gefei-shenzhen-share-20251213/kongxian.jpg)
这哥们的方法论特别厉害，底层逻辑很强。

比如月入万刀是哥飞社群的一个口号，下面是这哥们的拆解。

![月入万刀](/images/blog/xianxiahuodong/gefei-shenzhen-share-20251213/kongxian-million-dollar.png)

#### 核心思想与方法论
生活中许多事情都类似于徒步，一旦你开始行走，景色就会发生变化。

—— James Clear

+ 拥抱不确定性是成功的关键，需要先判断不确定性的来源（是否与自己相关），然后制定应对策略。
+ 人生是一个长期游戏，不确定性永远存在，需要一步步解决局部的不确定性。
+ 问题的质量决定答案的质量，好的问题需要足够细化，能够指引下一步行动。
+ 任何问题都存在"程度"的区别，不是非黑即白，而是有多种可能性和解决方案。
+ 实验思想：提出精准问题->设计实验->确定满意程度->得出结论。
+ 多维思考能帮助从不同角度看待问题，发现更多可能性。
+ 上下文链接的重要性，让用户自然流转到各个功能，提高转化率。

图论拆解--任何事情都是可以设计的。

![图论](/images/blog/xianxiahuodong/gefei-shenzhen-share-20251213/kongxian-graph.png)

#### 产品开发策略
+ 市场筛选：选择一个通用且老的赛道，使流量获取更加容易。
+ 调研方法
    - 纵向调研：通过拆的方式，不断做减法
    - 横向调研：解决方案是对某个问题的回答，我们只需要确定问题是什么，就能得到一个解决方案集合，从而实现产品/功能的横向调研
+ MVP开发：先做加法收集想法，然后问自己"要做到什么程度"来筛选必要功能。
+ 时间统筹：许多任务可以并行处理，如提交Strip的同时可以投外链。
+ 素材制作：初期可以简单处理，有流量后再优化；亲自制作素材能更好理解用户需求。
+ 产品定价：不要过分担心竞争对手的低价，因为用户不会花太多时间比较所有选项。

#### 用户沟通与客诉处理
+ 客诉是建立长期用户关系的机会，不仅是解决问题。
+ 处理退款请求时，先提供替代方案（如延长使用期限）再提供退款选项。
+ 与用户建立个人联系，了解他们的背景和需求。
+ 寻找机会与用户合作，如邀请他们成为种子用户或内容创作者。
+ "什么都能聊"的态度，换位思考，多维思考。

#### 总结
+ 面对不确定性，先判断它是否与自己相关，然后逐步解决局部问题
+ 将大问题拆解成具体问题，细化到能指导行动的程度
+ 进行多维思考，从不同角度看待产品和问题
+ 开发产品时，关注上下文链接，让用户自然流转到各个功能
+ 做产品调研时，不仅研究同类竞品，也要看不同领域如何解决类似问题
+ 处理客诉时，视为建立长期用户关系的机会，提供多种解决方案。

### 2.4 Niko-新人如何从0到1，赚到第一个1000美金
![Niko](/images/blog/xianxiahuodong/gefei-shenzhen-share-20251213/niko.jpg)
这位嘉宾也是大佬，我在朋友圈发了参加哥飞线下后，马上就有生财群友找到我，想要这位大佬分享的内容。

#### 出海踩过的一些坑
##### 第一坑：批量上站
在所有变量都不变的情况下，上1个站不赚钱，上10个站也不赚钱。

正确做法：

+ 高度聚焦，每上一个站都从各个角度归因，踩完所有的坑
+ 流量归因：
    - OnPage SEO、外链、内页
    - 停留时长、跳出率
+ 转化归因：
    - 定价设计、用户路径、转化漏斗
    - GA4流量分析、用户沟通

**产品盈利的两个核心点：流量和转化，搞通这两点就能赚钱。**

##### 第二坑：新手打王者局
接触的信息太多，把所有的路线都了解了个遍

新词一波流，老词反馈慢，做产品比较长久

在没有任何积累的情况下，直接跳过了两个路径选择去做产品

正确做法：

+ 新人不用考虑太多，先拿到正反馈
+ 在过程中收获的经验会帮你进入到下一个阶段
+ 跳出不赚钱的怪圈

#### 如何实现从0到月入千刀

通过不断迭代优化网站，提升用户体验和转化率。

##### 3.研究用户路径和转化漏斗
深入分析用户行为，优化转化路径。

##### 4.付费外链和投流
+ 首次获得年费订单：600刀收入，开始尝试广告投放，初期亏损
+ 付费外链初探：投入100刀到Toolify，收回200刀
+ 持续加大外链投入：直到 ROI 不再理想
+ 覆盖主流付费外链平台：Toolify，Creati.ai，Topai.tools

#### 全职出海和大厂上班的区别
+ 大厂上班：
    - 短期不焦虑，长期焦虑
    - 产品赚钱，与我无关
    - 资源多，限制多，效率低
    - 反馈来自年终奖和绩效，主观成分重
+ 全职出海：
    - 长期不焦虑，短期无反馈
    - 资源少，自由度大，效率高
    - 反馈来自用户，无法投机取巧

##### 如何在业余时间开启自己的副业
+ 唯一建议：做别人看得见的事情
+ 好处：
    - 主业自由：第一份工作自己找的，其他都是别人主动找我
    - 副业自由：接不完的单子，改不完的bug，代码能写到吐
    - 会有一些意外惊喜
